Laconic Configuration Specification

Laconic Configuration (Laconfig) is text data format like json or xml. Every laconfic file has root section with any number of subsections (named logical block delimeted with curly brackets) and attributes in it (named literal values)

root
{
  sec-a
  {
    attr1=1
    attr2='xyz'
  }

  sec-b
  {
    attr3=123
    sec-c {}
  }
}

Section and attributes names can be arbitrary string (even arbitrary Unicode character sequence).

Laconfic admits commenting of a single lines or line blocks in a c#-like manner

root
{
  age=32 // single line comment

  /* another one */

  /* 
    multiline 
    comment 
  */
}

An _override attribute is used for more sofisticated control over configuration section merge process. It has the following valid values:

  • attributes - attributes-only override
  • sections - sections-only override
  • all - sections and attributes override
  • replace - overriding section replaces base completely
  • stop - no section can modify anything in this one
  • fail - an exception is thrown when a child tries to override this section
root
{
  sec-a
  {
    _override='attributes'
    attr1=1
    attr2='xyz'
  }

  sec-b
  {
    _override='sections'
    attr3=123
    sec-c {}
  }
}
